# Architecture Decision Records

| # | Decision | Status | Summary |
|---|----------|--------|---------|
| 001 | Use OBO flows for device-local agent identity | Accepted | Agents get their own identity via OBO token exchange so audit logs attribute actions to the agent, not the human |

## ADR Format

Each ADR follows this structure:
- **Context**: Why we faced this decision
- **Decision**: What we chose
- **Rationale**: Why it's the right choice
- **Consequences**: Tradeoffs and implications
- **Alternatives Considered**: What we rejected and why
